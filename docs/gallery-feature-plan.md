# 갤러리 기능 구현 계획

## 개요

사용자가 AI 생성 이미지를 공유하고, 다른 사용자의 설정을 프리셋으로 가져와 활용할 수 있는 커뮤니티 갤러리 기능.

---

## 네비게이션

```
헤더 로고 클릭 (드롭다운)
├─ 풍경 모드    → /landscape
├─ 스튜디오 모드 → /studio
└─ 갤러리      → /gallery  ← NEW
```

---

## 핵심 기능

### 1. 갤러리 그리드
- 반응형 Masonry 레이아웃
- 썸네일 + 간단 정보 (좋아요 수)
- 무한 스크롤 또는 페이지네이션

### 2. 이미지 상세 보기 (라이트박스)
- 이미지 확대 보기
- 프롬프트/설정 정보 표시 (마치 실제 사진의 카메라 세팅 정보를 보는 것과 같은 느낌)
- 액션 버튼:
  - [이 설정으로 시작하기] → 해당 모드로 이동 + 설정 적용
  - [프롬프트 복사] → 클립보드 복사
  - [좋아요] → 좋아요 토글

### 3. 이미지 업로드 (관리자/사용자)
- 이미지 파일 업로드 **중요: 불건전하거나 잘못된 이미지를 어떻게 필터링할 것인지 검토 필요
- 프롬프트 텍스트 입력
- 생성된 AI 모델명 입력 ** 중요 **
- 모드 선택 (풍경/스튜디오)
- 설정값 JSON 입력 또는 자동 첨부

### 4. 좋아요 시스템
- 로컬스토리지 기반 (초기)
- 나중에 DB 연동 가능

### 5. 데이터 활용 기능 ** 중요 **
- 가장 많이 사용된 프롬프트 키워드 등을 추출해서, 향후 본 서비스의 개선을 위해 사용하는 방법 검토

## 데이터 구조

```typescript
interface GalleryItem {
  id: string;
  
  // 이미지 정보
  src: string;              // 이미지 URL
  thumbnail?: string;       // 썸네일 URL (optional)
  width: number;
  height: number;
  
  // 모드 및 설정
  mode: 'landscape' | 'studio';
  settings: LandscapeSettings | Settings;  // 전체 설정 객체
  prompt: string;           // 생성된 프롬프트 텍스트
  
  // 메타 정보
  title: string;
  description?: string;
  author?: string;
  createdAt: string;        // ISO string
  
  // 소셜
  likes: number;
}
```

## 사용 라이브러리

| 역할 | 라이브러리 | 버전 | 용도 |
|------|----------|------|------|
| 그리드 레이아웃 | `react-photo-album` | ^3.0 | 반응형 Masonry 그리드 |
| 라이트박스 | `yet-another-react-lightbox` | ^3.0 | 이미지 확대, 슬라이드쇼 |
| 라이트박스 플러그인 | `yet-another-react-lightbox/plugins/captions` | - | 캡션/정보 표시 |

### 설치 명령어
```bash
npm install react-photo-album yet-another-react-lightbox
```

## 파일 구조

```
app/
├─ gallery/
│  └─ page.tsx              # 갤러리 페이지

components/
├─ gallery/
│  ├─ GalleryGrid.tsx       # 그리드 레이아웃
│  ├─ GalleryLightbox.tsx   # 라이트박스 + 정보 표시
│  ├─ GalleryFilter.tsx     # 필터/정렬 UI
│  ├─ GalleryUpload.tsx     # 업로드 모달
│  └─ GalleryCard.tsx       # 개별 카드 컴포넌트

lib/
├─ gallery/
│  └─ gallery-data.ts       # 샘플 데이터 (JSON)

types/
├─ gallery.types.ts         # GalleryItem 타입 정의
```
